# 周日复盘流程（四步，顺序执行）

## 第 1 步 · 复盘

1. 读 `learning/ledger.md` 本周所有条目 + `learning/curriculum.md` 对照计划
2. 按 `learning/weekly-review.md` 模板逐项提问引导用户填写（复盘是用户的交付物，你提问、汇总数据，**不替他下结论**）
3. 特别追问"最反直觉的一个瞬间"——这是博客最值钱的素材
4. 填好的复盘追加到 weekly-review.md 末尾，commit

## 第 2 步 · 周记博客

1. 以复盘内容为素材起草周记（这一步可以代笔初稿，但观点/感受段落必须来自用户复盘原话）
2. 发布走 CLAUDE.md 的博客发布流程（~/jokeuncle.github.io，pnpm build 验证后 push）
3. 博客 URL 回填 ledger 作为本周证据

## 第 3 步 · 生成下周学习包

按 CLAUDE.md"周包工程师"角色执行：

1. 读 curriculum.md 下周骨架，创建 `week-NN-主题/` 目录，结构对齐 week-01：
   - README.md：学习地图（每日安排表）+ 必修知识点 + 必读资料链接
   - tools/：该周实验脚本，**每个脚本必须现场跑通才能入库**
   - notes.md：交付物骨架（只搭章节结构，正文留给用户）
2. 若 ledger 显示本周有未完成的学习日 → 下周计划头部插入补账安排，不静默跳过

## 第 4 步 · 细化题库

1. 把下周的验收题细化进 `learning/quiz-bank.md`（每个学习日 1-2 题，标注框架图节点，含追问）
2. **每周追加"面试题精选"小节**：该周主题下最高频的 5 道真实面试题（偏 AI Infra 推理方向岗位），每题给出：题目 + 追问方向 + 与本周哪个实验/笔记的关联。这些题同时充当验收题的进阶层
3. 全部 commit + push，一句话向用户汇报四步产出清单

## 红线

- 四步顺序不可乱：先复盘再写周记（否则周记没素材），先周包再题库（否则题没有载体）
- 工具脚本跑不通 → 不入库，宁可当场修
- 用户想跳过复盘直接要周包 → 提醒一次"复盘缺失会让周记断更"，用户坚持则尊重
