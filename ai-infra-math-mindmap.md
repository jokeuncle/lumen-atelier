## 1. 线性代数 ★贯穿全程

### 1.1 向量 🟢
- 是什么：一排数字，如 [1.2, -0.5, 3.1]
- 人话：LLM 里一个 token 就是一个向量（Qwen2.5-7B 里是 3584 个数字）
- 向量加法：对应位置相加 🟢
- 数乘：每个数都乘同一个系数 🟢
- 点积（内积）🟢：对应位置相乘再全部加起来，得到一个数
  - 人话：衡量两个向量"有多像"，attention 分数的本质就是 Q 和 K 做点积
  - 例子：[1,2]·[3,4] = 1×3 + 2×4 = 11

### 1.2 矩阵 🟢
- 是什么：数字表格，m 行 n 列
- 人话：LLM 的每个权重就是一张大表格，attn_q 是 3584×3584 的表格 ★W1 已见
- 转置 🟢：行列互换，记作 Wᵀ

### 1.3 矩阵乘向量 🟡（前置：点积）
- 规则：结果的第 i 个数 = 矩阵第 i 行与向量的点积
- 人话：把一个 3584 维向量"变换"成另一个向量——decode 阶段每生成一个 token 就是无数次这个操作
- ★为什么 decode 慢：矩阵每个数字只用一次就扔，搬运成本 > 计算成本

### 1.4 矩阵乘矩阵（GEMM）🟡（前置：矩阵乘向量）
- 规则：C = A×B，C 的第 i 行第 j 列 = A 第 i 行 与 B 第 j 列的点积
- 形状规则 🟢：[M,K] × [K,N] → [M,N]，中间的 K 必须相等（背下来！）
- 人话：prefill 阶段整个 prompt 一起算，就是矩阵乘矩阵，GPU 最喜欢
- FLOPs = 2×M×K×N 🟡：每个输出要 K 次乘法 + K 次加法
- ★读 vLLM 源码时 90% 的形状检查靠"中间维度必须相等"这一条

### 1.5 张量与形状演算 🟡（前置：矩阵）★W1 正在练
- 是什么：矩阵再叠一层就是 3D 张量，如 [batch, seq, hidden]
- 人话：想象一摞 Excel 表格，batch 是第几本、seq 是第几行
- reshape 🟢：总元素数不变，重新分组，如 [2,6] → [3,4]
- transpose 🟡：交换维度顺序，[B,S,H] → [B,H,S]
- stride（步长）🔴（前置：reshape）：ne[]/nb[] ★ggml 已见
  - 人话：数据在内存里是一条直线，stride 说"跳几个数字算换一行"
  - 为什么重要：transpose 可以只改 stride 不搬数据（零拷贝）
- 广播 broadcasting 🟡：小张量自动"复制"去匹配大张量的形状

### 1.6 特殊矩阵结构 🟡
- 单位矩阵 🟢：对角线全 1，乘了等于没乘
- 对角矩阵 🟢：只有对角线有值 → 计算超便宜
- 正交/旋转矩阵 🟡（前置：点积）：
  - 人话：只旋转不拉伸的变换，二维就是 [[cosθ,-sinθ],[sinθ,cosθ]]
  - ★RoPE 就是把 Q、K 向量按位置转不同角度——高中三角函数够用
- 低秩分解 🔴（前置：GEMM）：
  - 人话：一张 3584×3584 的大表格 ≈ 3584×8 乘 8×3584 两张小表格
  - 这就是 LoRA：微调只训小表格，参数量降 200 倍
  - 想深究最优性 → SVD（可跳过，用到再学）

## 2. 数值计算与浮点 ★Infra 的差异化数学

### 2.1 二进制与位 🟢
- bit：一个 0 或 1；byte = 8 bits
- 人话：Q4 量化 = 每个权重只用 4 个 bit ≈ 半个 byte ★W1 的 4.4GB 就是这么来的
- 2 的幂速查：2¹⁰=1024≈千(K)，2²⁰≈百万(M)，2³⁰≈十亿(G)

### 2.2 科学计数法 → 浮点数 🟡（前置：二进制）
- 高中版：6.02×10²³，一个"有效数字"部分 + 一个"指数"部分
- 浮点数 = 二进制的科学计数法：符号位 + 指数位 + 尾数位
- FP32 = 1+8+23 位 🟡
- FP16 vs BF16 🟡：
  - 人话：总共 16 位的预算，FP16 给"精度"多分位，BF16 给"范围"多分位
  - BF16 范围同 FP32 → 训练不容易炸（溢出）
- FP8 的 E4M3/E5M2 🟡：同样的取舍再来一次

### 2.3 浮点误差 🟡（前置：浮点数）
- 舍入误差 🟢：0.1 在二进制里是无限循环小数，存不精确
- 大数吃小数 🟢：1000000.0 + 0.0001 = 1000000.0（小数被"吃"了）
- 灾难性抵消 🟡：两个几乎相等的大数相减，有效数字全没了
- ★浮点加法不满足结合律：(a+b)+c ≠ a+(b+c) → GPU 并行求和每次结果微差 → 这是"同 seed 不同输出"的元凶之一

### 2.4 数值稳定性技巧 🔴（前置：浮点误差 + 指数函数）★读源码常见
- 指数函数复习 🟢：e^x 增长极快，e^100 就溢出 FP32 了
- softmax 🟡：把一排分数变成一排"概率"（全正、和为 1）
  - 公式：每个数取 e^x 再除以总和
  - 人话：放大差距的"选秀打分"——分高的概率占大头
- 减 max 技巧 🟡：softmax(x) = softmax(x − max)，数学上完全相等
  - 人话：所有分数先集体减去最高分，最大值变 0，e^0=1 永不溢出
- online softmax 🔴（前置：减 max 技巧）★W3 前必懂：
  - 人话：数据分批到来，每批更新"目前最大值"和"目前总和"，不用等全部到齐
  - 这一个技巧 = FlashAttention 能省显存的全部数学
- ★RMSNorm 保 F32 的原因：平方求和容易溢出/失精 ★W1 已见 141 个 F32 张量

### 2.5 量化数学 🔴（前置：浮点数 + 四舍五入）★W1-W2 正在碰
- 核心思想 🟢：把连续的小数"塞进"有限的格子里
  - 人话：身高 150-190cm 的人只发 S/M/L/XL 四种码的衣服
- 均匀量化 🟡：q = round(x/s)，s 是"每格多宽"（scale）
  - 例子：权重都在 -1~1 之间，4 bit 有 16 个格子，每格宽 2/16 = 0.125
- zero-point 🟡：格子不对称时的偏移量
- per-tensor / per-channel / group-wise 🟡：
  - 人话：整件衣服一个尺码 vs 每个部位单独量尺寸——越细越准、元数据越多
- Q4_K super-block 🔴（前置：均匀量化）★W1 已拆：
  - 人话：256 个权重一组，组内再分 8 小组，小组各有 scale——就是"部位单独量"
  - 这就是为什么 Q4_K 实际是 4.5 bit/权重，多的 0.5 是尺码表本身
- 量化误差从哪来 🟡：格子太宽（舍入）+ 个别超大权重把格子撑宽害了大家（outlier）
- GPTQ/AWQ 🔴（前置：以上全部；GPTQ 还要一点微积分直觉）：
  - 人话：GPTQ 问"哪些权重糙点没事"；AWQ 问"哪些通道是 VIP 要保精度"
  - 会用即可，推导可跳过

## 3. 概率与信息论 ★W2 用到

### 3.1 概率分布 🟢
- 是什么：每个可能结果配一个概率，加起来 = 1
- 人话：LLM 每一步输出的本质 = 对 152064 个 token 的一张"彩票概率表" ★W1 vocab
- 期望 🟢：加权平均，中彩票的"平均收益"

### 3.2 采样的数学 🟡（前置：概率分布 + softmax）★W2 实验直接用
- logits 🟢：还没变成概率的"原始分数"
- temperature 🟡：softmax(x/T)
  - 人话：T 小 → 分数差距放大 → 学霸通吃（保守）；T 大 → 差距抹平 → 人人有机会（发散）
  - T→0 = 永远选最高分（贪心）
- top-k 🟢：只保留分数前 k 名，重新分配概率
- top-p 🟡：从高到低累加概率直到超过 p，之后的全砍掉
  - 人话：top-k 是"只看前 40 名"，top-p 是"只看累计占 95% 的头部"——后者自适应
- 重复惩罚 🟢：出现过的 token 分数打折

### 3.3 信息论三件套 🔴（前置：概率分布 + 对数）
- 对数复习 🟢：log 把乘法变加法；log(1)=0；概率越小 −log 越大
- 熵 🟡：平均"惊讶程度"。骰子比硬币熵大（更难猜）
- 交叉熵 🟡：用错误的概率表去猜，平均要多惊讶多少 → LLM 训练损失就是它
- KL 散度 🟡：两张概率表的"差异分数" → 衡量量化前后模型输出变化
- perplexity 🔴（前置：交叉熵）★W2 要跑：
  - 公式：exp(平均交叉熵)
  - 人话：模型每一步"平均在几个候选里犹豫"。PPL=8 ≈ 平均 8 选 1 的不确定度
  - 量化实验里 PPL 涨 = 模型变笨了多少的量尺

### 3.4 投机解码的数学 🔴（前置：3.2 + 3.3）★W7+，现在可跳过
- 接受概率 min(1, p大/p小)：小模型抢答，大模型批卷
- 数学保证：最终分布与大模型独立采样完全相同（无损）

## 4. 微积分与优化 ·推理岗浅涉即可

- 导数 🟢：变化率，切线斜率（高中内容）
- 链式法则 🟡：复合函数求导逐层相乘 → 反向传播的全部数学
- 梯度 🟡：多变量版导数，指向"上坡最陡"的方向；训练 = 往反方向走
- Adam 🔴（前置：梯度）：带"惯性"和"自适应步长"的下坡法——概念了解即可
- loss scaling 🟡：FP16 梯度太小会变 0（下溢），先放大再缩回
- ★推理方向到此为止，别在这里陷太深

## 5. 算法与复杂度 ★工程日常

### 5.1 大 O 记号 🟢
- 人话：输入变大时，耗时/内存怎么涨。O(n)=翻倍涨，O(n²)=翻倍变四倍
- 只看最高次项：n² + 100n 就叫 O(n²)

### 5.2 attention 的复杂度账 🟡（前置：大 O + GEMM 形状）
- Q[n,d] × Kᵀ[d,n] → 分数矩阵 [n,n] → O(n²) 显存，O(n²d) 计算
- 人话：seq 翻倍，attention 矩阵变 4 倍大 → 长上下文贵的根源
- FlashAttention：分块 + online softmax → 不存整个 n² 矩阵 → 显存降回 O(n)

### 5.3 KV cache 的账 🟡（前置：乘法就够了）★W1 作业，现在就算
- 公式：2(K和V) × layers × n_kv_heads × head_dim × seq × 每数字字节数
- 代入 Qwen2.5-7B @ seq=2048, FP16：
  - 2 × 28 × 4 × 128 × 2048 × 2 bytes = **117MB**
- 人话：没 GQA（28 头全存）就是 7 倍 = 820MB；100 个并发用户 = 11.7GB → PagedAttention 为什么必须存在
- 无 cache 每步重算全部历史 = O(n²)；有 cache 每步只算新 token = O(n)

### 5.4 分页的数学 🟡（前置：除法+取整）★W3-W4
- 人话：KV cache 按"页"分配（如 16 token 一页），跟操作系统内存分页同一招
- 内碎片：最后一页平均浪费半页；页越大浪费越多、页表越小——权衡
- prefix 共享：多个请求同 system prompt → 共享前缀页 → 树结构（RadixAttention）

### 5.5 排队论一行公式 🟢
- Little's Law：并发数 = 吞吐 × 延迟
- 人话：奶茶店同时在做 10 杯，每杯 2 分钟 → 每分钟出 5 杯。容量规划就这一行

## 6. 性能建模 ★面试最爱，优先级最高

### 6.1 两个"速度上限" 🟢
- 算力（FLOPS）：每秒做多少次乘加
- 带宽（GB/s）：每秒从显存搬多少字节
- 人话：厨师切菜速度 vs 传菜员搬菜速度，慢的那个决定出餐速度

### 6.2 算术强度与 Roofline 🟡（前置：6.1）
- 算术强度 = 计算量 ÷ 搬运量（FLOPs/byte）
- 人话：每搬一斤菜能炒几盘？强度高 = 厨师是瓶颈（compute-bound）；低 = 传菜员是瓶颈（memory-bound）
- decode：每个权重搬进来只用 1 次 → 强度 ≈ 1 → 永远 memory-bound ★你的 25 t/s
- prefill：整个 prompt 复用同一批权重 → 强度 ≈ seq 长度 → compute-bound
- batch=B 的 decode：权重搬 1 次用 B 次 → 强度 ×B → continuous batching 的存在理由

### 6.3 心算公式（背下来）🟢 ★必须滚瓜烂熟
- 模型大小：参数量 × 每参数字节（7B×Q4≈0.5byte ≈ 3.5GB + 元数据 → 实测 4.4GB ✓ ★W1）
- decode 上限 ≈ 带宽 ÷ 模型字节：M5 Pro ~150GB/s ÷ 4.4GB ≈ 34 t/s（你实测 25 → 效率 73%）
- TTFT ≈ prompt 长度 × prefill 单 token 耗时；TPOT ≈ 1000ms ÷ decode t/s
- KV cache：用 5.3 的公式
- 这五条能心算，Infra 面试的估算题基本通关

### 6.4 MFU 与 scaling law 🟡
- MFU = 实际达到的 FLOPS ÷ 理论峰值：训练 40-50% 算优秀
- scaling law 🟢：loss 随参数量按"幂律"下降——知道曲线形状即可

## 7. 并行计算的数学 ★W9+ 云端周，现在可整章跳过

- Amdahl 定律 🟢：10% 的部分死活不能并行 → 无论多少卡最多加速 10 倍
  - 人话：一道菜必须最后由主厨亲自摆盘，帮厨再多也快不过摆盘时间
- allreduce 通信量 🟡：环形算法每卡收发 ≈ 2×数据量，与卡数基本无关
- TP 每层 2 次 allreduce 🟡 → 通信太频繁 → 不出节点（NVLink 内解决）
- PP 气泡率 🔴（前置：流水线直觉）：(p−1)/(m+p−1)，microbatch 越多气泡越小
- α-β 模型 🟡：通信耗时 = 固定延迟 + 数据量÷带宽（快递 = 打包时间 + 运输时间）
